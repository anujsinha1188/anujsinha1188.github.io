Extra | Java Data Types | Primitive Data Types

  • Java is statically-typed (must declare variable before use)
  • 8 Primitive Data Types:
    • byte - 8-bit signed two's compliment integer | Default value 0 | Range-2^7 to 2^7-1
    • short - 16-bit signed two's compliment integer | Default value 0 | Range -2^15 to 2^15-1
    • int - 32-bit signed two's compliment integer | Default value 0 | Range -2^31 to 2^31-1 (0 to 2^32 in unsigned in Java 8)
    • long - 64-bit two's compliment integer | Default value 0L | Range -2^63 to 2^63-1 (0 to 2^64-1 in unsigned in Java 8)
    • float - single-precision 32-bit IEEE 754 floating point | Default value 0.0f | Do not use for precise value - use java.math.BigDecimal for precise value (currency)
    • double - double-precision 64-bit IEEE 754 floating point | Default value 0.0d | Do not use for precise value
    • boolean - size isn't precisely defined | Default value false
    • char - single 16-bit Unicode character | Default value \u0000 | Range \u0000 (0) to \uffff (65,535)
    • String - NOT primitive data type | Default value null | enclosing character string in double quotes create new String object | String objects are immutable.
  • Literals - A literal is the source code representation of a fixed value (int a = 1)
  • Integer Literals - recommended to use upper case L as lower case letter l is hard to distinguish from digit 1 | Can be expressed in:
    • Decimal, base 10 (int a = 1)
    • Hexadecimal, base 16 | 0x indicates hexadecimal (int a = 0x1)
    • Binary, base 2 | 0b indicates binary (int a = 0b1)
  • Floating-Point Literals - type float ends with F/f | type double ends with D/d (optional) | Can also be expressed with scientific notation E/e | double d1 = 1.234e2D
  • Character and String Literals
    • char and String may contain Unicode(UTF-16) character.
    • can use a "unicode escape" such as '\u0108'
    • special escape sequence - \b (backspace) | \t (tab) | \n (line feed) | \f (form feed) | \r (carriage return) | \" (double quote) | \' (single quote) | \\ (backslash)
    • class literal - formed by taking a class name and appending ".class" (String.class) | refers to the object (of type class) that represent the type itself
  • Using Underscore Character in Numeric Literals - In Java SE 7 and later, any number of underscore characters (_) can appear anywhere between digits in a numerical literal. (improve the readability of code)

Object-Oriented Java

  • Instance Variables - save those specific details we want the class to include.
  • The void keyword (which means "completely empty") indicates to the method that no value is returned after calling that method.
  • Class: a blueprint for how a data structure should function
  • Constructor: instructs the class to set up the initial state of an object
  • Object: instance of a class that stores the state of a class
  • Method: set of instructions that can be called on an object
  • Parameter: values that can be specified when creating an object or calling a method
  • Return value: specifies the data type that a method will return after it runs
  • Inheritance: allows one class to use functionality defined in another class

Extra | Object-Oriented Java | Object-Oriented Programming Concepts

  • What is an Object?
    • Two characteristics: state (variables) and behaviour (functions)
    • Hiding internal state and requiring all interaction to be performed through an object's methods is known as data encapsulation
    • Benefits of Bundling code into individual software objects: Modularity, Information-hiding, Code re-use, Pluggability and debugging ease
  • What is a Class?
    • A class is a blueprint or prototype from which objects are created.
    • In object-oriented terms, we say that created object is an instance of the class of objects.
  • What is Inheritance?
    • Object-oriented programming allows classes to inherit commonly used state and behavior from other classes.
    • In the Java programming language, each class is allowed to have one direct superclass (using extends), and each superclass has the potential for an unlimited number of subclasses
  • What is an Interface?
    • Interfaces form a contract between the class and the outside world, and this contract is enforced at build time by the compiler.
    • When a class implements an interface, it promises to provide the behavior published by that interface.
  • What is a Package?
    • A package is a namespace that organizes a set of related classes and interfaces.
    • The library (provided by java platform)is known as the "Application Programming Interface", or "API" for short.
    • Java Platform API Specification: contains the complete listing for all packages, interfaces, classes, fields, and methods supplied by the Java SE platform.

Extra | Java Access Control | Controlling Access to Members of a Class

  • Access level modifiers determine whether other classes can use a particular field or invoke a particular method.
  • There are two levels of access control:
    • At the top level—public, or package-private (no explicit modifier).
    • At the member level—public, private, protected, or package-private (no explicit modifier).
    • The following table shows the access to members permitted by each modifier.
    • Access Levels
    • Modifier Class Package Subclass World
      public Y Y Y Y
      protected Y Y Y N
      no modifier Y Y N N
      private Y N N N
