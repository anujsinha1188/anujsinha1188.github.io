    What are Directives?

  • directives are markers on a DOM element (such as an attribute, element name, comment or CSS class) that tell AngularJS's HTML compiler ($compile) to attach a specified behavior to that DOM element (e.g. via event listeners), or even to transform the DOM element and its children.
  • What does it mean to "compile" an HTML template? For AngularJS, "compilation" means attaching directives to the HTML to make it interactive. The reason we use the term "compile" is that the recursive process of attaching directives mirrors the process of compiling source code in compiled programming languages.
  • Normalization - AngularJS normalizes an element's tag and attribute name to determine which elements match which directives.
  • The normalization process is as follows:
    • Strip x- and data- from the front of the element/attributes.
    • Convert the :, -, or _-delimited name to camelCase.
  • Directive types
  • Creating Directives - directives are registered on modules. To register a directive, you use the module.directive API. module.directive takes the normalized directive name followed by a factory function | The function is invoked using $injector.invoke
  • When you create a directive, it is restricted to attribute and elements only by default.
  • The restrict option is typically set to: 'A' | 'E' | 'C' - >!-- directive: my-dir exp --< | 'M' - >span class="my-dir: exp;"<>/span<
  • These restrictions can all be combined as needed: 'AEC'
  • When should I use an attribute versus an element? Use an element when you are creating a component that is in control of the template. Use an attribute when you are decorating an existing element with new functionality.
  • Isolating the Scope of a Directive - separate the scope inside a directive from the scope outside, and then map the outer scope to a directive's inner scope.
  • the isolate scope of the directive isolates everything except models that you've explicitly added to the scope: {} hash object
  • The scope option is an object that contains a property for each isolate scope binding.
  • a scope prototypically inherits from its parent. An isolated scope does not.
  • Creating a Directive that Manipulates the DOM - link option to register DOM listeners as well as update the DOM. | signature - function link(scope, element, attrs, controller, transcludeFn) | AngularJS scope object - jqLite-wrapped element that this directive matches - hash object with key-value pairs of normalized attribute names and their corresponding attribute values - directive's required controller instance(s) - a transclude linking function pre-bound to the correct transclusion scope.
  • When a DOM node that has been compiled with AngularJS's compiler is destroyed, it emits a $destroy event. Similarly, when an AngularJS scope is destroyed, it broadcasts a $destroy event to listening scopes.
  • signature: element.on('$destroy', ...) | scope.$on('$destroy', ...)
  • transclude makes it so that the contents of a transcluded directive have whatever scope is outside the directive
  • & binding allows a directive to trigger evaluation of an expression in the context of the original scope, at a specific time.
  • pass data from the isolate scope via an expression to the parent scope: passing a map of local variable names and values into the expression wrapper function.
  • require: '^^myTabs' - The ^^ prefix means that this directive searches for the controller on its parents. (A ^ prefix would make the directive look for the controller on its own element or its parents; without any prefix, the directive would look on its own element only.)
  • If it is necessary to reference the controller or any functions bound to the controller from the template, you can use the option controllerAs to specify the name of the controller as an alias
  • use controller when you want to expose an API to other directives. Otherwise use link.
