INTERMEDIATE JAVASCRIPT MODULES

  • JavaScript modules are reusable pieces of code that can be exported from one program and imported for use in another program.
  • The pattern to export modules:
    • Define an object to represent the module.
    • Add data or behavior to the module.
    • Export the module. | module.exports = ObjectToExport
  • The pattern to import a module:
    • Import the module. | const VariableName = require(PathToImport)
    • Use the module and its properties within a program.
  • default export | default export ObjectToExport | module.exports = ObjectToExport
  • import | import VariableName from PathToImport | const VariableName = require(PathToImport)
  • Named Export | allow us to export data through the use of variables.
    • export { nameOFObjToExport, nameOfObjToExport2 };
    • import { nameOFObjToExport, nameOfObjToExport2 } from './jsFileName'
    • Named exports can be exported as soon as they are declared, by placing the keyword export in front of variable declarations.
  • Export as | Named exports also offer a way to change the name of variables when we export or import them with as keyword.
  • Import as | An object exported could be aliased with the as keyword when imported. | Another way of using aliases is to import the entire module as an alias | import * as nameOfObject
